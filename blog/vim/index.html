VIM EDITOR

Mastering VIM: A Comprehensive Tutorial

VIM, short for "Vi IMproved," is a highly configurable text editor built to enable efficient text editing. It's a command-line based editor, meaning it doesn't rely on a graphical interface like other text editors such as Sublime Text or VSCode. VIM is pre-installed on most Unix-based systems (Linux, macOS), and it's available for Windows as well.


Basic Navigation

Moving the Cursor:

KeyPurpose
h, j, k, lMove the cursor left, down, up, and right respectively.
w Move forward to the beginning of the next word.
b Move backward to the beginning of the previous word.
e Move to the end of the current word.

Jumping between Lines:

KeyPurpose
0Move to the beginning of the current line.
^Move to the first non-blank character of the line.
$Move to the end of the current line.
ggMove to the beginning of the file.
GMove to the end of the file.

Scrolling:

KeyPurpose
Ctrl + uScroll up half a screen.
Ctrl + dScroll down half a screen.
Ctrl + bScroll up one full screen.
Ctrl + fScroll down one full screen.

Editing Text

Entering Insert Mode:

KeyPurpose
i Enter insert mode before the cursor.
a Enter insert mode after the cursor.
I Enter insert mode at the beginning of the line.
A Enter insert mode at the end of the line.
o Open a new line below the current line and enter insert mode.
O Open a new line above the current line and enter insert mode.

Exiting from any Mode:

KeyPurpose
Esc Exit insert mode and return to normal mode.

Deleting Text:

KeyPurpose
x Delete the character under the cursor.
dw Delete from the cursor to the beginning of the next word.
d$ Delete from the cursor to the end of the line.
dd Delete the current line.
D Delete from the cursor to the end of the line

Undo and Redo:

KeyPurpose
u Undo the last change.
Ctrl + r Redo the last undone change.

Saving and Quitting

Saving:

KeyPurpose
:w Save changes to the file.
:wq Save changes and quit.
:x Save changes and quit (same as :wq).
ZZ Save changes and quit (if changes have been made).

Quitting without Saving:

KeyPurpose
:q! Quit without saving changes.
:qa! Quit all open files without saving changes.

Advanced Techniques

Searching:

KeyPurpose
/search_term Search forward for search_term.
?search_term Search backward for search_term.
n Move to the next occurrence of the search term.
N Move to the previous occurrence of the search term.

Copying, Cutting, and Pasting:

KeyPurpose
yy Copy the current line.
dd Cut the current line.
p Paste the copied or cut text after the cursor.
P Paste the copied or cut text before the cursor.

Registers

KeyPurpose
"[regNum]yyYank (copy) text into register [regNum].
"[regNum]pPaste text from register [regNum].
:regDisplay contents of all registers.
:reg [regNum]Display contents of a specific register.

Splitting Screen:

Anything starting with ':' is a command. After typing the command press RETURN to see the changes. Also the split screen you can quit from the split screen of your choice by changing the focus to the desired split screen and then using the normal saving and quitting commands to exit.
KeyPurpose
:split [FILENAME]split horizontally with [FILENAME].
:vsplit [FILENAME]split vertically with [FILENAME].
Ctrl + w + ARROW_KEYMove focus to desired splitted screen in the given direction.
Ctrl + w + < Decrease the width of the split (vertical split).
Ctrl + w + > Increase the width of the split (vertical split).
Ctrl + w + - Decrease the height of the split (horizontal split).
Ctrl + w + + Increase the height of the split (horizontal split).
Ctrl + w + w Cycle forward through open splits.
Ctrl + w + W Cycle backward through open splits.

Visual Mode

Visual mode in Vim is a powerful feature that allows you to select text for editing, copying, cutting, or formatting. When you enter Visual mode, you can visually highlight text using movement commands. Once you've selected the desired text, you can perform various operations on it.

Entering and exiting:

KeyPurpose
vEnter Visual mode character-wise.
VEnter Visual mode line-wise.
Ctrl + vEnter Visual block wise. Select in a rectangular block shape by moving the cursor.
EscPressing Esc exits Visual mode without performing any operation.

Operations in Visual Mode:

KeyPurpose
yYank (copy) the selected text.
dDelete (cut) the selected text.
~Toggle case of the selected text.
**>Indent the selected text.
<Unindent the selected text.
cChange (replace) the selected text.

Advanced Visual Mode Shortcuts:

KeyPurpose
gvReselect the last visual selection.
oMove the cursor to the other end of the selected text.
UConvert the selected text to uppercase.
uConvert the selected text to lowercase.
:sPerform a substitution on the selected text.

Vimrc File

These are the configuration for making the vim better. Anything written after " is a comment.
.vimrc 

"set number " uncomment to get line number
set nocompatible
set ts=4 sw=4 smartindent "setting up indenation
set incsearch "autosearch while trying to search using / or ?
color elflord "setting bg color
syntax on "syntax highlighing

"shift+(Up/Down) shifts the line up/down
"works on single and multiple line
nnoremap <S-Up> :m .-2<CR>==
vnoremap <S-Up> :m '<-2<CR>gv==gv
nnoremap <S-Down> :m .+1<CR>==
vnoremap <S-Down> :m '>+1<CR>gv==gv

"Alt+(Up/Down) makes a copt moves the line up/down
"wont work for multiple lines in visual mode
nnoremap <A-Up> :t-<CR>
vnoremap <A-Up> :t'<-CR>gv==gv
nnoremap <A-Down> ddPp
vnoremap <A-Down> ddPp

"To change the position of a character to left or right
nnoremap <A-Left> x<Left>P
nnoremap <A-Right> xp

" Enable auto completion menu after pressing TAB.
set wildmenu
" Make wildmenu behave like similar to Bash completion.
set wildmode=list:longest
" There are certain files that we would never want to edit with Vim.
" Wildmenu will ignore files with these extensions.
set wildignore=*.docx,*.jpg,*.png,*.gif,*.pdf,*.pyc,*.exe,*.flv,*.img,*.xlsx

" STATUS LINE ------------------------------------------------------------ {{{
" Clear status line when vimrc is reloaded.
set statusline=
" Status line left side.
set statusline+=\ %F\ %M\ %Y\ %R
" Use a divider to separate the left side from the right side.
set statusline+=%=
" Status line right side.
set statusline+=\ ascii:\ %b\ hex:\ 0x%B\ row:\ %l\ col:\ %c\ percent:\ %p%%
" Show the status on the second to last line.
set laststatus=2
" }}}

" Preventing Ctrl+z to stop vim abruptly
nnoremap <C-z> <Esc>

" Ctrl + /
nnoremap <C-_> : call CommentToggle()<CR>
" Ctrl + / (visual mode)
vnoremap <C-_> : call CommentToggle()<CR>

" function to toggle comments in c,cpp,python.
function! CommentToggle()
    let l:filetype = &filetype
    let l:line = getline('.')
    let l:trimmed_line = substitute(l:line, '^\s*\(.\{-}\)\s*$', '\1', '')

    if (l:filetype ==# 'c' || l:filetype ==# 'cpp' || l:filetype ==# 'java' || l:filetype ==# 'javascript' || l:filetype==# 'css')
        if l:trimmed_line[:1] !=# '/*'
            execute "normal! I/* <Esc>A */<Esc>"
        else
            execute "normal! ^xxx$xxx"
        endif
    elseif (l:filetype ==# 'python' || l:filetype ==# 'sh')
        if (l:trimmed_line[:1] !=# '# ')
            execute "normal! I# <Esc>"
        else
            execute "normal! ^xx<Esc>"
        endif
    elseif (l:filetype ==# 'html' || l:filetype ==# 'xml')
        if l:trimmed_line[:3] !=# '<!--'
            execute "normal! I<!-- <Esc>A --><Esc>"
        else
            execute "normal! ^xxxxx$xxxx"
        endif
    endif
        echo "fileType: " . l:filetype
endfunction
